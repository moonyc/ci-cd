GitHub Actions

Key Elements and First Workflow

GitHub Actions is a workflow automation service offered by GitHub:
it can automate all kind of possible repository processes.
We need to understand three fundamental building blocks to efficiently integrate
the service in our pipelines: workflows, jobs and steps.
Workflows, jobs and steps are related to each other and to the code repository like this:

  1. The workflows are attached to the repositories.
    You can add as many workflows as you want to a given repository.
  2. Workflows contain one or more jobs.
  3. Jobs contain one or more Steps
  4. Steps define the actual commands that will be executed.

We build workflows to automate processes, consequentially
we assign a trigger (or event) to each workflow, so that we're able to regulate
their execution's recurrence. An event could require manual activation,
but you could also trigger the a workflow whenever a commit is pushed to a certain branch.
