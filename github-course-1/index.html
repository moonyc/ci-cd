Let's say you want to store the code of your projects on GitHub.
The first thing that you want to do is creating a remote repository.
You will target this repository from your local environment, so that
you will be able to create a stable pipeline from the directory you're working on
to the code that has been stored on the cloud. In your local environment:

  1. Make sure you have git installed or install it
  2. Open the terminal and check that the path matches the directory's (example: /Users/user/your-project)
  3. git init -> initialize git in your project
  4. git status -> verify the current status of the files in the directory
  5. git add file.pl or git add . -> the first stages the designated file
    the second stages all the files in the directory
  6. if you now run git status again you'll see that bunch of files are ready to be committed
  7. git commit -m "meaningful description of the current state of the project" ->commits are a stored version of you code base
  8. git log -> this command returns a list of commits in the current branch:
    a branch, in git, is a pointer to a specific commit
  9. press q to exit the git log output
  10. git checkout commit_id -> move temporarely to another commit. In other words it lets you
    access a precedent state of your code base
  11. now your current commit is detached from the original branch
    so if you want to come back to your latest commit, you must execute git checkout name_of_the_branch
  12. git reset --hard id_commit -> here the id indicates the destination you want to move your project's state to,
    eradicating all the changes made after.

Now you know some of the most important and the most common commands of git.
At this point what you want to do is to finally push your code on GitHub and eventually organize
different and parallel collections of commits, each dedicated to a specific path or a specific feature.
In git an independent collection of commits is a branch. At initialization you commit on the master
branch, and you rename it by executing the command git branch -M main, which is a standard, I guess.
Storing your codebase on GitHub is quite simple: first you create a repository on github.com and then
in your local environment:

  1. git remote add origin https://github.com/ yourHandle/ name-of-the-repository.git
    -> target your repository online from the local environment
  2. git push -u origin main -> push your commits on the cloud.
    the -u flag indicates that you're setting an upstream branch, which connects your local main branch to one with the same name in the cloud. If on github you don't have a main branch, this command will automatically create it.
  3. git remote add origin https://youHandle@ github.com/ yourHandle/ name-of-the-repository.git -> if you get a permission error, this command will let you authenticate. On github.com's account settings, create a Personal Access Token and paste it in the password field. You can also set up the ssh key to authenticate.

At this point you might wanna take some time to develop a specific feature,
while avoiding to influence the current state of your project. This is how you do it in
your local environment:

  1. git branch new-features-branch-name -> create a new branch with the latest
    state of your project as a starting point. You can develop the new feature in this new branch without modifying the state of the main branch. When the new feature will reach a stable state, you will be able to reintegrate the changes.
  2. git merge new-features-branch -> integrate the the new branch and the target branch into
    a single collection; execute the command from the target branch (ex: from main). You might need to solve a couple of inconsistencies manually.
  3. git branch -> shows all the exsting branches
  4. git checkout target-branch -> switch to the target branch.
    You can use the same command to switch back
  5. git branch -D target-branch -> delete the target branch
  6. git checkout -b new-target-branch -> create a new branch and checkout in one step
  7. git push origin new-features-branch-name -> push your commits into a new branch on GitHub
